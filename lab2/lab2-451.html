2. Lab Questions

Complete the following problems; you must use the following template and submit your assignment on Canvas to receive credit. For each question, list all sources/websites used and students consulted.

  1. First, run and test Listing 3-1 on your ESP32; the debounce.ino program on pages 59-63 of our course textbook. Next, run and test Listing 3-2 on your ESP32; the press.ino program on pages 63-67 of our course textbook. In your own words, explain the queue behavior in Listing 3-2 versus Listing 3-1; namely, how are the producer(s) and consumer(s) different, how is the data different, etc.
  2. Run and test Listing 3-3 on your ESP32; the press2.ino program on pages 68-71 of our course textbook. In your own words, explain why this is a safety improvement over Listing 3-2 and what change(s) were made in the code to implement this.

Once you can run this program successfully, use a private repository on git to commit the original code, then modify it so that the blink period is half a second and commit your modified code. Be sure to use a private repository so that your code is not publicly accessible. Once you have tested your modified program, copy-paste (as text) your code from this modified program as your response to this question.
  4. Submit a screenshot of the commit view of your private git repository for the modification you made to the code in question 3. This screenshot should show your commit with comment/message, as well as a comparison (i.e., diff) between the original code and your modified code. Use the separate Canvas submission link to submit this.
  5. Run and test Listing 4-2 on your ESP32; this is the delayuntil.ino program on pages 91-94 of our course textbook. The two LEDs do not continue to blink in unison; in your own words, explain which LED blinks first and why this delay occurs.
